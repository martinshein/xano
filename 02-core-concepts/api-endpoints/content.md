Content 
=======

The Metadata API enables you to interact with database table content (database records). The following are various of examples of how to create, update, delete, and truncate content.

###  

Create Content

In this example, we will create content. The endpoint requires a workspace ID and table ID. In addition, the name of the field(s) and value(s) that you wish to create.

![](../../_gitbook/image3a28.jpg?url=https%3A%2F%2F3699875497-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F2tWsL4o1vHmDGb2UAUDD%252Fuploads%252F53YMgyiw1EEjzp97y49r%252FCleanShot%25202023-04-12%2520at%252018.30.40%25402x.png%3Falt%3Dmedia%26token%3D160ab631-3a72-4dd3-8844-6d6de9de0578&width=768&dpr=4&quality=100&sign=1ec7462d&sv=2)

Example response body:

Copy

``` 
{
  "id": 11,
  "created_at": 1681349436618,
  "name": "headphones",
  "description": "high quality listening device",
  "category_id": 3,
  "price": 99
}
```

###  

Update Existing Content

To update an existing record, the content ID (primary ID of the record) is also required. In this example, we will update the price from the above example to be 500

The Metadata is flexible with updates: We only need to pass what we want to update. For example, if we want all the other fields to remain the same for ID 11 but want to update price then we can just pass the new value for price.

![](../../_gitbook/imaged83f.jpg?url=https%3A%2F%2F3699875497-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F2tWsL4o1vHmDGb2UAUDD%252Fuploads%252FlwrubxpPJqNs14aUiv19%252FCleanShot%25202023-04-12%2520at%252018.36.27%25402x.png%3Falt%3Dmedia%26token%3D4df38e44-c36f-4d79-b880-250a9e29fd77&width=768&dpr=4&quality=100&sign=2f23db8d&sv=2)

Example response body:

Copy

``` 
{
  "id": 11,
  "created_at": 1681349436618,
  "name": "headphones",
  "description": "high quality listening device",
  "category_id": 3,
  "price": 500
}
```

###  

Delete Content

Deleting content is straightforward, it requires the content ID, table ID, and workspace ID.

![](../../_gitbook/image29fe.jpg?url=https%3A%2F%2F3699875497-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F2tWsL4o1vHmDGb2UAUDD%252Fuploads%252FpwrYsyFdxPIn1RIvihiW%252FCleanShot%25202023-04-12%2520at%252018.37.59.png%3Falt%3Dmedia%26token%3Df9cd604b-4027-40a2-94b1-befa8b74a8c6&width=768&dpr=4&quality=100&sign=e01cb110&sv=2)

The response will be null since we are just deleting a record.

###  

Truncate Content

Truncate content will clear all the content of a table. You can optionally reset the primary key back to 1 for the table.

![](../../_gitbook/image6e2f.jpg?url=https%3A%2F%2F3699875497-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252F2tWsL4o1vHmDGb2UAUDD%252Fuploads%252F8SUCMtmnoHIKPx0M6Y5V%252FCleanShot%25202023-04-12%2520at%252018.41.30.png%3Falt%3Dmedia%26token%3D58d8c415-0703-48e5-a733-25328efbc326&width=768&dpr=4&quality=100&sign=b38176d5&sv=2)

The response will be null since we are truncating the table.

This action cannot be done and will result in the loss of content (records).
